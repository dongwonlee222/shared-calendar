# Shared Calendar Spec

## 개요

이 캘린더는 `공용 업무 캘린더`다.

- 목적: 여러 프로젝트 일정을 한 화면에서 보고, 월/주 기준으로 운영한다.
- 사용 형태: 브라우저 기반 단일 페이지
- 현재 소스 구조:
  - 앱: [index.html](/Users/dongwon.lee/workhub/company/shared/calendar/index.html)
  - 초기 데이터: [calendar-data.js](/Users/dongwon.lee/workhub/company/shared/calendar/calendar-data.js)
  - 파비콘: [favicon.svg](/Users/dongwon.lee/workhub/company/shared/calendar/favicon.svg)

## 현재 기능

### 1. 보기

- `월 / 주` 전환
- `Month` 선택 + 이전/다음 달 이동
- `Week` 선택 + 이전/다음 주 이동
- `오늘` 플로팅 버튼으로 현재 월/현재 주 이동
- 주간 보기에서 현재 시간선 표시

### 2. 프로젝트 필터

- 프로젝트별 on/off
- `미분류` 일정 별도 on/off
- 색상은 프로젝트 기준

### 3. 일정

- 일정 클릭 시 상세 팝업 표시
- 기존 일정 수정/삭제 가능
- 주간 시간 일정:
  - 드래그로 날짜/시간 이동 가능
  - 위/아래 핸들로 길이 조절 가능
- 월간/주간에서 날짜 선택 후 새 일정 등록 가능

### 4. 백로그

- 우측 `백로그` 탭에서 메모 입력
- 백로그별 프로젝트 선택 가능, 기본은 `프로젝트 미선택`
- 백로그 항목 드래그 가능
- 드롭 규칙:
  - 월간 날짜칸 드롭: 24시간 일정
  - 주간 24시간 업무 줄 드롭: 24시간 일정
  - 주간 시간칸 드롭: 1시간 일정

### 5. 아젠다

- 현재 보고 있는 월/주 범위의 일정 목록 요약
- 달력은 배치 보기, 아젠다는 리스트 보기 역할

## 데이터 구조

### 초기 데이터

[calendar-data.js](/Users/dongwon.lee/workhub/company/shared/calendar/calendar-data.js) 안의 `window.CALENDAR_DATA`

- `projects`
  - 프로젝트 메타
  - 예: 이름, 색상
- `events`
  - 초기 일정 데이터

예시 필드:

```js
{
  id: "ec-clean-qa-field",
  project: "environment-corp",
  tag: "tentative",
  owner: "클린일렉스",
  title: "현장 QA",
  start: "2026-04-20",
  end: "2026-04-24",
  status: "조율 필요",
  myDates: ["2026-04-22"],
  note: "4/22 참석 예정."
}
```

### 브라우저 저장 데이터

`localStorage`

- 사용자 생성 일정
- 백로그 항목

즉:

- 초기 기준 데이터 = `calendar-data.js`
- 사용자 작업 데이터 = 브라우저 저장

## 태그 규칙

- `manufacturer_schedule`
  - 제조사 일정
- `project_milestone`
  - 프로젝트성 마일스톤
- `review`
  - 검토 일정
- `tentative`
  - 미확정/조율 필요 일정

태그는 표시 우선순위와 표현 차이에 쓰인다.

## 주의할 점

### 1. 전체 render 최소화

현재 캘린더는 기능이 많아서 `render()` 전체 호출이 많아지면 스크롤이 튈 수 있다.

우선 원칙:

- 단순 선택 = class 토글
- 범위 드래그 표시 = class 토글
- 진짜 레이아웃이 바뀌는 경우만 `render()`

### 2. 주간 뷰 스크롤

- 주간 시간영역은 `24시간 업무` 아래만 스크롤된다.
- 리사이즈/이동 후 스크롤 위치가 유지되어야 한다.
- 이 부분을 깨는 수정은 우선적으로 피한다.

### 3. 백로그 프로젝트

- 백로그의 프로젝트는 `생성 시점`에 항목에 저장된다.
- 드롭 시점 선택값으로 덮어쓰지 않는다.

### 4. 미분류 일정

- 프로젝트가 없는 일정은 `미분류`로 표시된다.
- 항상 보이게 두지 않고 필터로 on/off 한다.

## 사용자 요청 문구 예시

캘린더 쪽 작업 내용을 알려고 할 때는 이렇게 말하면 된다.

- `캘린더 현재 구조 설명해줘`
- `캘린더 스펙 기준으로 정리해줘`
- `shared/calendar 기준으로 뭐가 구현돼 있는지 알려줘`
- `캘린더에서 최근에 바뀐 기능 요약해줘`
- `캘린더 수정 전에 구조부터 확인해줘`

캘린더를 수정하고 싶을 때는 이렇게 말하면 된다.

- `캘린더에서 주간 뷰만 손보자`
- `shared/calendar에서 백로그 동작 수정해줘`
- `공용 캘린더 스펙 기준으로 일정 등록 UX 바꿔줘`
- `캘린더 렌더 최적화 계속 진행해줘`
- `캘린더 작업 이어서 해줘`

## 운영 제안

앞으로는 캘린더 관련 요청을 할 때 아래 중 하나를 항상 붙이는 걸 추천한다.

- `shared/calendar 기준`
- `공용 캘린더 기준`
- `캘린더 스펙 기준`

이렇게 하면 다른 프로젝트 작업과 섞이지 않는다.
